# 내일은행 — 기능명세서
> **"사라진 동네 지점장을, AI로 복원하다."** — AI 챗봇이 아니라 **AI 지점장.**

### JB금융그룹 Fin:AI Challenge · 자유주제(JB금융그룹 연계 AI Agent) · 팀명: 〔팀명〕
> MVP 제안서와 동일 서비스(정합). 3p 권장 분량. 〔구성도·흐름도는 발표용 다이어그램으로 별도 시각화 권장 — 아래는 그 설계.〕

## ① 서비스 개요 (Service)

**비전:** *내일 들어올 확정 현금흐름을 오늘.* 지역화폐를 시작점으로 카드·배달 등 확정채권으로(보험·지원금·급여는 *규제·적격성 검토* 후 — ⑤ 2축 등급).

**쉽게:** 오늘 매출은 확정인데 정산금은 며칠 뒤 — *AI 지점장*이 그 돈이 들어올 걸 미리 알고, 은행원 승인을 거쳐 *먼저* 지급한다.

**내일은행**은 광주은행이 *운영*하는 지역화폐(광주상생카드) 정산의 **'시간차'를 메우는 AI Agent('AI 지점장')** 서비스다. 은행원 업무화면에서 작동하며, *확정된* 정산 채권을 **기계적 산식(정산예정액×고정 haircut)으로 선지급**하고 — AI는 *위험 경보·정책 매칭·거절 연착륙*으로 보좌 — 정산일 회수까지 자율 관리한다. (개별 신용대출이 아닌, **시장 검증된 확정채권 담보 단기여신**)

- **사용자:** 은행원(주 사용자, 코파일럿) · 소상공인(수혜자)
- **핵심 가치:** 흑자도산 방지 유동성 · 저위험 단기수익 · 부실 사전관리
- **범위:** *새 금융상품 출시가 아니라* **AI Agent 구현**이 MVP 목표(서비스화 시 팩토링·선정산·혁신금융 등 제도 검토).
- **2층:** *쐐기* = 지역화폐 정산(광주은행 운영·해자, MVP). *플랫폼* = [확정채권] 카드·배달·보험·지원금 / [조건부·규제검토] 급여 EWA·ATM 생활비.
- **원칙:** 선지급 **한도=기계 산식·자격=네거티브 스크리닝**(AI는 한도 게이트 아님) → AI는 *위험 발견·추천·설명*, **최종 승인은 규칙엔진 + 은행원**(고영향 AI 인간 최종판단)

## ② 시스템 구성도 (Architecture)

```
[1] 데이터 계층
    정산예정(운영대행)·입출금·상권정보 공공API·정책공고문
    (MVP/예선: 합성 정산데이터 + 상권정보 공공API)
        |
        v
[2] 전처리·피처
    정산 스케줄 정규화 · 회수확률 피처 · 반복매출/거래처 · 상권·축제 시즌
        |
        v
[3] 멀티 에이전트 코어  ── 오케스트레이터가 전문 에이전트를 조율 (판단->행동->검증)
    +-------------------------------------------------------+
    | 오케스트레이터 (워크플로우 제어 · 도구 호출 · 결과 종합)     |
    +-----------+-----------+-----------+-------------------+
    | 회수확률    | 위험       | 정책       | 수익성             |  <- 전문 에이전트
    | 에이전트    | 에이전트   | 에이전트   | 에이전트            |   (각자 결정론
    | GBM       | 이상탐지   | RAG+룰    | 한도·금리 최적화     |    모델·도구)
    +-----------+-----------+-----------+-------------------+
            | 각자 점수·위험플래그·근거를 '구조화 출력'으로 제출
            v
    +-------------------------------------------------------+
    | 규칙엔진 = 중재: 충돌 조정·한도·금리캡·정책 검증·집행       |  <- 결정론 최종
    | LLM = 각 에이전트 근거를 SHAP 기반 자연어로 '설명만'       |   (무환각·감사)
    +-------------------------------------------------------+
        |
        v
[4] 행동·연동
    은행원 코파일럿 UI · 위험 알림 · 규칙엔진 승인 · 정산일 자동 상계 회수
        |
        v
[5] 검증·개선
    회수 결과 피드백 -> 모델 재보정 · 임계값 갱신 · 감사 로그
```

- **멀티 *전문* 에이전트:** 오케스트레이터가 *회수확률·위험·정책·수익성* 전문 에이전트(각자 결정론 모델·도구)를 호출·조율하고, 규칙엔진이 충돌을 중재·집행. → **단일 룰이 아니라 *목적함수가 다른 전문 판단들의 협업·조정*이라, "왜 그냥 룰 기반이면 안 되나"에 답한다.**
- **결정론/LLM 분리:** *판단(승인·한도·위험)은 결정론 모델·규칙엔진*, LLM은 *설명·검색·대화*만 → 환각을 판단 경로에서 차단(감사 가능). — *대회 지정주제2가 명시한 '규칙기반+LLM 결합' 방향과 동일.*
- **규제 가드(규칙엔진 하드코딩):** 선지급 한도=정산예정액×고정 haircut(채권평가감)·금리/수수료 ≤ 법정상한·**haircut과 이자 분리**·네거티브 스크리닝 — 전부 결정론 규칙으로 강제(감사 가능).
- **실행 구조(서비스화 단계):** *AI 판단·추천은 실행과 무관하게 동작.* 자금·채권 처리는 **1순위 광주은행 자기자금 여신**(담보=확정채권, 금리 법정상한 하드캡, haircut·이자 분리) — *증권사 매도대금 선지급이 검증한 담보부 단기여신 구조*. **JB우리캐피탈 팩토링**은 자금조달·위험분산 옵션(채무자≠매입자 = 그룹 해자, *"이자 회피"용 아님*). 〔구체 구조는 서비스화 시 확정〕

## ④ 주요 기능 흐름도 (Flow) — Agent Loop

```
(트리거) AI가 매일 밤 관할 소상공인 능동 스캔 -> 위험·기회 발견 -> 은행원에 먼저 알림
        (또는 정산예정 발생 / 은행원 조회)
   |
   v  ── 판단 ──
정산 '확정' 검증(스크리닝) -> 한도=정산예정액×haircut(기계) + 회수확률·위험 플래그(라우팅용)
   |
   v  ── 검색(도구 호출) ──
정책 RAG: 해당 소상공인 자격 정책 매칭
   |
   v  ── 생성 ──
추천 한도 + SHAP 근거 + 정책 결과를 패키징
   |
   v  ── 행동 ──
규칙엔진 검증(스크리닝·사전승인 정책·한도)
   -> [스크리닝 통과·무플래그] 자동 집행(선지급)     (Agent의 행동)
   -> [위험 플래그·고위험] 은행원 에스컬레이션 -> 승인/보류
   -> (거절 시) F2 정책·F3 연착륙 자동 호출         (거절-연착륙)
   * AI 단독 자동거절 아님 + 계약이행 단서·거부권·설명·인적재심 보장(§37의2)
   |
   v  ── 검증/개선 ──
정산일 자동 회수 -> 결과를 모델에 피드백
   -> (정상) 종료      -> (이상) 재시도·위험경보(F3)
```

핵심: *데이터 → 검색 → 판단 → 생성 → 검증 → 후속 액션*이 하나의 자율 루프로 연결(공통요건 충족).

## ⑤ 향후 발전 방향 (Future Work)

- **실현 로드맵:** **Phase 0** 합성데이터 PoC(데모) *완료* → **Phase 1** 광주은행 실데이터 소규모 파일럿(회수확률 실학습·계약/규제 정리·은행원 베타) → **Phase 2** 정식 출시 + 캐피탈 팩토링 구조 확정 → **Phase 3** 확장(카드·배달 → 적격성 검토 후 보험·지원금 → 계열사).
- **★키스톤:** *운영대행 계약상 선지급 허용 + 소상공인 명시 동의 취득 권한(정산데이터 여신활용·자동상계).* 이 하나가 풀리면 — 기술 보유·캐피탈 그룹내·데모 동작 — 나머지는 빠르다.
- **에이전트 고도화:** (코어의 전문 에이전트를) 동적 워크플로우·자율 협상으로 확장, 수익성↔위험 에이전트 간 trade-off를 회수 결과로 학습.
- **확장(2축 등급):** [확정·실행적격] 지역화폐(MVP·키스톤)→카드·배달 / [조건부] 보험금·정부지원금(지원금 법정 양도금지 多)→개별검토 / [부적격·별도 규제] 급여 EWA·ATM(임금직접지급·채권부존재). 지리적: 광주→전남 22개 시군→JB금융그룹(전북은행).
- **규제 경로:** **(1순위) 은행 고유 여신**(재원 자기자금, 담보=확정채권, 금리 법정상한 하드캡, haircut·이자 분리) — *증권사 매도대금 선지급이 검증한 단순·견고 구조*. (옵션) **JB우리캐피탈 매출채권 팩토링**(채무자≠매입자, 캐피탈 본업) = 자금조달·위험분산 수단(*"이자 회피"용 아님*, 진정양도 적격성 법률검토). 자동실행·신상품화 시 **혁신금융서비스 지정** 검토. 정산데이터 여신활용은 **명시적 동의 기반 실명 처리**(*가명처리 아님* — 개별 식별 한도산정은 정의상 실명 처리).

## ⑥ 부록 (Appendix)

- **기술 스택:** Python · **LightGBM+SHAP**(회수확률) · scikit-learn(이상탐지) · 규칙엔진(정책·금리캡·haircut 코드화) · **온프레미스 Gemma 12B 계열**(vLLM·4bit 양자화, 설명·대화 *전용*) · RAG(한국어 임베딩+pgvector) · 자체 오케스트레이터 · 웹 UI. *LLM은 판단 안 함 → 12B로 충분, 정산데이터 외부 반출 0.*
- **데이터 출처/라이선스:** 소상공인시장진흥공단 상권정보 공공 API(data.go.kr, 오픈) · 합성 정산/거래 데이터(자체 생성) · 〔실데이터는 광주은행 운영 대행 데이터, 계약·규제 검토 필요〕
- **MVP 검증(실증):** **동작 데모 `내일은행_데모.html`** — 합성 정산데이터로 **haircut 기계 한도**(정산예정액×고정률) + 멀티에이전트 4종 위험판단 → 3단계 신호 → SHAP → 자동집행/거절-연착륙까지 *실제 동작*(4개 케이스). `[구현]` 기능이 화면에서 살아 움직임을 확인.
- **용어:** 운영 대행사(지역화폐 보관·판매·환전·정산 대행 금융기관) · 시간차 브릿지(확정 채권의 선지급) · 규칙엔진(승인·한도 정책의 결정론적 검증).
- **단위경제(가정·예시, 파일럿서 교체):** 평균 200만/18일·**연환산 수수료율 7~9%**(건당 ≈0.4%) → 건당 마진 작음. 회전은 ROA 아닌 **ROE 레버리지** / **net ROA는 자동화율이 좌우**(출시초기 30~50% → 목표 90%+; haircut으로 손실노출 축소).
